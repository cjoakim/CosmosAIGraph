What is CosmosAIGraph?

  • CosmosAIGraph is not a Azure Cosmos DB Database or API
  • Rather, it is a set of working solutions or Reference Implementations that customers may copy and modify
  • See the CosmosAIGraph Public GitHub repository
  • These demonstrate how to implement AI-powered Graph Workloads and Knowledge Graphs with Azure Cosmos DB
  • CosmosAIGraph is applicable to a broad array of industries, companies, and use cases
  • Its goal is to provide low-cost and scalable solutions that integrate into the Azure platform
  • This page focuses on the first solution; others will follow

Improvements over its' predecessor, AltGraph

  • Adds a W3C standards-based graph query language
  • Adds a support for W3C standards-based well-defined schema or ontology
  • Adds Artificial Intelligence (AI) with Azure OpenAI
  • Adds Vector Search with Azure Cosmos DB
  • No "triples" documents are required. The graph is loaded from selected attributes of your domain documents instead
  • See the AltGraph Blog

Technical Foundations

  • The Cosmos DB Mongo vCore API is the operational datastore. It supports vector search
  • The implementation programming language is Python3
  • CosmosAIGraph is influenced by LinkedIn, which leverages an in-memory graph for both high-performance and lower costs
  • A guiding design principle is to use Open Standards and Open-Source software where appropriate, specifically:
    • The Resource Description Framework (RDF) and W3C standards
    • The RDF Web Ontology Language (OWL)
    • The SPARQL 1.1 RDF query syntax
    • The LangChain AI framework for developing applications powered by language models
    • Docker containerization for simple and high-quality deployments
    • The FastAPI web framework, with OpenAPI webservice definition support
  • CosmosAIGraph is deployed as a set of three microservices - Web, Graph, and AI
  • The primary deployment target is Azure Container Apps (ACA). It can also be deployed to Azure Kubernetes Service (AKS)
  • The deployment process uses the Bicep domain-specific language (DSL) and modern declarative DevOps syntax
